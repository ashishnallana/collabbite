# CollabBite

**Real-Time Collaborative Food Ordering Experience.**

CollabBite acts as a collaboration layer on top of Swiggy. It allows groups (friends, colleagues, families) to browse restaurants, add food items to a shared real-time cart, chat with a built-in HUD, and eventually allows the host to push the entire order back to Swiggy for checkout.

## Architecture

CollabBite is designed as a highly responsive, modern multi-tier web application built with a **Dockerized** microservices approach.

- **Frontend**: Next.js (App Router) React application featuring Glassmorphism UI, real-time HUD overlays, and responsive mobile design.
- **Backend**: Express.js Node server providing REST APIs and WebSocket (Socket.io) multiplexing.
- **Primary Database**: PostgreSQL (accessed via Prisma ORM) for persisting session data, user roles, and cart states.
- **In-Memory Cache & PubSub**: Redis is used to store highly ephemeral data like the Live Chat history, bypassing PostgreSQL entirely for speed and scale.
- **Background Jobs**: Node-Cron runs scheduled sweeps every 10 minutes to auto-delete sessions older than 3 hours, keeping the PostgreSQL DB clean.
- **Swiggy MCP Integration**: The backend natively integrates with Swiggy MCP via stateless HTTP POST requests, using OAuth 2.0 PKCE authentication.

## Production Deployment (Docker Compose)

The entire application is fully dockerized for a production-ready setup.

### Prerequisites
- Docker Engine & Docker Compose

### 1. Build and Run
From the root directory, simply run:
```bash
docker-compose up -d --build
```
This single command will:
1. Pull and spin up a **PostgreSQL 15** container.
2. Pull and spin up a **Redis 7** container.
3. Build the **Backend** image (running `npx prisma generate` and `tsc`).
4. Build the **Frontend** Next.js image.
5. Network everything together.

**Ports Exposed**:
- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:5000`
- PostgreSQL: `5432`
- Redis: `6379`

## Local Development (Without Docker)

### Prerequisites
- Node.js (v18+ recommended)
- PostgreSQL running locally (e.g., via WSL or local installer)
- Redis running locally on port 6379

### 1. Backend Setup
1. `cd backend`
2. `npm install`
3. Create `.env`:
   ```env
   DATABASE_URL="postgresql://<user>:<password>@localhost:5432/collabbitedb?schema=public"
   REDIS_URL="redis://localhost:6379"
   PORT=5000
   ```
4. `npx prisma db push && npx prisma generate`
5. `npm run dev`

### 2. Frontend Setup
1. `cd frontend`
2. `npm install`
3. `npm run dev`

## Workflow & Features

1. **Host Creates Session**: 
   The Host authenticates via Swiggy OAuth directly in the browser. The backend connects directly to Swiggy's MCP server to fetch the host's addresses, creating a session in PostgreSQL with a strict **3-hour TTL**.
2. **Guests Join**: 
   Guests join the session using the unique ID generated by the Host. Socket.io automatically propagates their presence in real-time. Local storage is strictly scoped to the `sessionId` to prevent tab collisions.
3. **Collaboration Layer**: 
   - **Shared Cart**: Users can search for nearby restaurants and add menu items. Updates sync instantly.
   - **Activity Ticker**: Real-time notifications ("User added Burger to cart") slide in over the UI ephemerally without hitting the database.
   - **Live HUD Chat**: A video game-style HUD chat overlay allows users to talk and send floating emojis (`👍, 🍔, 🤤, 🔥`). All chat is powered purely by **Redis List** cache (`rpush/lrange`) for maximum throughput.
4. **Checkout**: 
   When all guests toggle "Ready", the Host clicks "Place Order". The backend parses the PostgreSQL cart into the Swiggy Cart schema format and pushes it to the Host's Swiggy account.

## Tech Stack
- **Frontend**: Next.js, React, Vanilla CSS (Glassmorphism), Socket.io-client, Axios, Lucide-React.
- **Backend**: Express, Socket.io, Node-Cron, Prisma ORM.
- **Databases**: PostgreSQL (Stateful Data), Redis (Ephemeral Chat/PubSub).
- **Integrations**: Swiggy MCP Server (Direct Stateless HTTP JSON-RPC Handshake).
- **Infra**: Docker, Docker Compose.
