# CollabBite

**Real-Time Collaborative Food Ordering Experience.**

CollabBite acts as a collaboration layer on top of Swiggy. It allows groups (friends, colleagues, families) to browse restaurants, add food items to a shared real-time cart, and eventually allows the host to push the entire order back to Swiggy for checkout.

## Project Structure

This project is a monorepo consisting of:
- **`frontend/`**: Next.js (App Router) application.
- **`backend/`**: Express.js server, PostgreSQL via Prisma, and Socket.io for WebSockets.

---

## Setup & Installation

### Prerequisites
- Node.js (v18+ recommended)
- PostgreSQL running locally or in the cloud.
- [swiggy-cli](https://github.com/swiggy/swiggy-cli) installed and authenticated on the Host's machine. The backend directly interfaces with `swiggy-cli` to fetch live data. 

### 1. Backend Setup

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Environment Setup: Create a `.env` file in the `backend/` directory:
   ```env
   # Example PostgreSQL Connection String
   # The database name we are using is collabbitedb
   DATABASE_URL="postgresql://<user>:<password>@localhost:5432/collabbitedb"
   ```
4. Push Prisma Schema (creates tables if they don't exist):
   ```bash
   npx prisma db push
   ```
5. Start the backend development server:
   ```bash
   npm run dev
   ```
   *The server runs on `http://localhost:5000`.*

### 2. Frontend Setup

1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the frontend development server:
   ```bash
   npm run dev
   ```
   *The application runs on `http://localhost:3000`.*

## How It Works

1. **Host Creates Session**: The Host starts a session on the frontend. The backend queries `swiggy-cli` to fetch the host's Swiggy addresses and creates a session in PostgreSQL.
2. **Guests Join**: Guests join the session using the unique ID generated by the Host. Socket.io automatically propagates their presence in real-time.
3. **Adding Food**: Users can search for nearby restaurants and add menu items to a shared cart.
4. **Checkout**: When ready, the Host clicks "Place Order". The backend parses the PostgreSQL cart into the Swiggy Cart schema format, executes the CLI to push it to the Host's Swiggy account, and the Host pays via the Swiggy app.

## Tech Stack
- **Frontend**: Next.js, React, Vanilla CSS (Glassmorphism), Socket.io-client, Axios, Lucide-React.
- **Backend**: Express, Socket.io, Prisma ORM, PostgreSQL, `@prisma/adapter-pg`.
- **Integrations**: Swiggy MCP Server (`swiggy-cli`).
